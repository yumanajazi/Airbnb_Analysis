🏠 Airbnb Data Analytics Project
The Goal
I took a raw, "messy" dataset of Airbnb listings and transformed it into a clear, interactive dashboard. The objective was simple: Help a potential host or investor understand where the money is, what guests want, and how prices vary by neighborhood.

🛠️ Tools Used
SQL (SQLite): Used for the heavy lifting—cleaning data and running complex queries.

DB Browser: My environment for managing the database.

Power BI: Used to build the final interactive visuals and maps.

🧼 The "Dirty" Work (Data Cleaning)
Before the charts could look good, the data had to be fixed. I used SQL to:

Fix Typos: Cleaned up neighborhood names that were spelled incorrectly.

Convert Money: Changed price columns from text (like "$120") into actual numbers so I could calculate averages.

Handle Gaps: Managed missing values in reviews and host info to keep the insights accurate.

💡 What can this Dashboard do?
The final Power BI Dashboard answers three major questions:

Where is the best place to host?

An interactive map highlights "hotspots" where revenue potential is highest.

How should I price my room?

I analyzed price trends based on the type of room (Entire Home vs. Private Room) and location.

What drives demand?

Charts showing the link between host response times, guest reviews, and booking frequency.

📈 Key Project Insights
The Pricing Gap: Found that certain neighborhoods are underpriced despite having high demand.

Supply Trends: Most listings are for entire homes, showing a gap in the market for budget-friendly private rooms in specific areas.
